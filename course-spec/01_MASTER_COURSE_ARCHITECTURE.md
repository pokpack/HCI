# Human-Computer Interaction (HCI)
## Master Course Architecture Version 1.0

## 1. รายละเอียดรายวิชา
- ชื่อรายวิชา: Human-Computer Interaction (HCI)
- กลุ่มเป้าหมาย: นักศึกษาวิศวกรรมคอมพิวเตอร์ ชั้นปีที่ 4
- Hands-on Learning 80%
- Theory 20%
- Project-Based Learning
- User-Centered Design
- Engineering-Oriented HCI
- 15 คาบเรียน คาบละ 3 ชั่วโมง
- เรียนทุกวันจันทร์
- ภาคการศึกษา: 22 มิถุนายน 2569 – 12 ตุลาคม 2569

## 2. โครงสร้างรายวิชา
GUI → Voice → Gesture → AR → VR → Multimodal Interaction

## 3. หลักการสำคัญ
1. ทุกบทต้องมี Prototype
2. ทุก Prototype ต้องมี Usability Report
3. ประเมินจาก Effectiveness, Efficiency, Satisfaction
4. ทุกโครงงานต้องเชื่อมโยงกับงานวิศวกรรมจริง

## 4. ตารางภาพรวมรายวิชา
(อ้างอิง Master Schedule ที่ยืนยันร่วมกัน)

## 5. โครงสร้างบทเรียน
1. พื้นฐาน HCI และ Usability Engineering
2. UX Research และการวิเคราะห์ผู้ใช้
3. การออกแบบ UI และสร้างต้นแบบด้วย Figma / AI Tool
4. การปฏิสัมพันธ์ด้วยเสียง (Voice Recognition)
5. การปฏิสัมพันธ์ด้วยท่าทาง (Gesture Recognition)
6. เทคโนโลยีความเป็นจริงเสริม (AR)
7. เทคโนโลยีโลกเสมือน (VR)
8. การบูรณาการเทคโนโลยีและโครงงานปลายภาค

## 6. Learning Flow
ศึกษาผู้ใช้ → ออกแบบ → พัฒนาต้นแบบ → ทดสอบ → Usability Report → ปรับปรุง

## 7. Deliverables
- GUI Prototype + Usability Report
- Voice Prototype + Usability Report
- Gesture Prototype + Usability Report
- AR หรือ VR Prototype + Usability Report
- Final Integrated Project
- Complete Usability Report

## 8. Success Criteria
- ออกแบบระบบเชิง User-Centered Design ได้
- สร้าง Prototype ได้
- ประเมิน Usability ได้
- จัดทำ Usability Report ได้
- ประยุกต์ใช้ Voice, Gesture, AR และ VR ได้
- พัฒนาโครงงาน HCI เชิงวิศวกรรมได้ครบวงจร
